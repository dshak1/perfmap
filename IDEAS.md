Good System To Reference
Use something simple and believable.

Best options:

Request-scoped dedup / enrichment service

A request arrives with many repeated IDs
You build a temporary map for that request
Then discard/reset it
Easy to explain
Per-batch metadata cache

A batch of events/documents comes in
You build a temporary lookup table for just that batch
Then clear it and process the next batch
This matches your scratch workload well
Graph traversal visited map

Each BFS/DFS query builds temporary visited state
Then resets for the next query

For the current winning niche, I would use:

“Per-batch document enrichment cache.”

Simple explanation:

each batch contains many records
records reference large metadata blobs
we build a temporary lookup structure for that batch
after the batch finishes, we clear and rebuild
that is where ScratchIndirectHashMap wins
